# public

## Purpose

Static assets served at the site root by Next.js. Anything in here is reachable at `/<filename>` without bundling.

## Key Files

| File | Description |
|------|-------------|
| `favicon.ico` | Browser favicon |

## Subdirectories

| Directory | Purpose |
|-----------|---------|
| `demo/` | Demo screenshots / videos referenced from the landing page (generated by `frontend/scripts/save-demo.js`) |
| `images/` | Static image assets used across landing and workspace pages |

## For AI Agents

### Working In This Directory

- Files here are **publicly reachable**. Do not commit secrets, tokens, or unredacted user data.
- Optimize images before committing (WebP / AVIF preferred for photos; SVG for icons).
- Large binary assets (>1 MB) should go through Git LFS or be hosted externally.

### Common Patterns

- Reference assets in code as `/<path>` (Next.js serves `public/` at the root).
- Use `next/image` for responsive image rendering when consuming these assets.

## Dependencies

### Internal

- Consumed by `src/components/landing/` and `src/components/workspace/`
